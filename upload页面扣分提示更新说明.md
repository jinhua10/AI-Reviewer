# Upload页面扣分提示更新说明

## 📋 更新内容

在 `upload.html` 页面添加了新的**AI评审扣分项提醒**信息框，帮助参赛者了解哪些行为会影响AI评分。

---

## 🎯 设计原则

### ✅ 已遵守的原则

1. **去除敏感信息**：不透露具体扣多少分
2. **精简语言**：用简洁的要点列表
3. **保持客观**：只说明事实，不主观评判
4. **提供建议**：给出改进方向

### ❌ 避免的内容

- ❌ 不透露具体扣分数值（如"-10分"）
- ❌ 不透露评分计算公式
- ❌ 不透露各维度的权重
- ❌ 不透露总分范围

---

## 📝 添加的提示内容

### 🚨 AI评审扣分项提醒

新增了一个**红色警示框**，包含以下内容：

#### 1️⃣ **安全问题**
```
代码中硬编码API密钥、密码、token等敏感信息
```

**说明**：
- 基于42分报告中的"硬编码API密钥在5个文件"问题
- 去除了"-10分"的具体扣分信息
- 提醒这是严重的安全问题

#### 2️⃣ **代码质量**
```
缺少单元测试文件、无日志系统(只用print)、异常处理过于宽泛(catch Exception)
```

**说明**：
- 综合了"无测试"、"无日志"、"通用异常"三个问题
- 去除了具体扣分（-9分、-3分、-2分）
- 用括号补充说明（如"只用print"）

#### 3️⃣ **创新性**
```
仅组合现有工具库而无独特改进或创新
```

**说明**：
- 基于"直接使用库"、"组合工具"的评分标准
- 去除了"基础分6-8分"的具体信息
- 强调需要有独特价值

#### 4️⃣ **实用性**
```
依赖付费云服务、部署复杂、缺少Docker支持
```

**说明**：
- 综合了AWS Bedrock依赖、部署难度等问题
- 去除了具体扣分（-6分）
- 提示部署便利性的重要性

#### 5️⃣ **文档完整性**
```
缺少依赖文件(requirements.txt等)、部署文档不完整
```

**说明**：
- 基于"无requirements.txt"等问题
- 强调工程化文档的重要性

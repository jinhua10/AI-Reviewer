# Writer Palmyra 超级严格评分配置指南

## 🎯 配置目标
让 Writer Palmyra 模型的打分变得**超级严格**，倾向给更低的分数和更严格的评价。

---

## ⚙️ 已应用配置（极度严格模式）

```yaml
temperature: 0
top-p: 0.3  # 极度严格模式 ✅
```

---

## 📊 参数详解

### Temperature（温度）

**当前值：0** ✅ 已是最严格

| 值 | 效果 | 评分行为 |
|----|-----|---------|
| 0 | 完全确定性 | 每次评分完全一致，最严格标准 |
| 0.1 | 极低随机性 | 几乎一致，略有波动 |
| 0.5 | 中等 | 评分会有明显变化 |
| 1.0 | 高随机性 | 评分差异大，不稳定 |

**结论：temperature: 0 是最严格的，不需要改动。**

---

### Top P（Nucleus Sampling）

**修改：0.9 → 0.3** 🔥 大幅降低

| Top P 值 | 覆盖概率 | 评分严格度 | 说明 |
|---------|---------|-----------|-----|
| **0.1-0.3** | 前 10-30% | **⭐⭐⭐⭐⭐ 极度严格** | **🔥 倾向选择最保守、最严格的评价** |
| 0.4-0.5 | 前 40-50% | ⭐⭐⭐⭐ 非常严格 | 标准高，给分谨慎 |
| 0.6-0.7 | 前 60-70% | ⭐⭐⭐ 适度严格 | 比默认严格 |
| 0.8-0.9 | 前 80-90% | ⭐⭐ 正常 | 平衡严格度和灵活性 |
| 1.0 | 100% | ⭐ 最宽松 | 考虑所有可能性 |

**为什么 top-p: 0.3 更严格？**

- 只考虑概率最高的前 30% token
- 排除了那些"宽容"、"高分"的表达
- 倾向选择"有问题"、"需要改进"等严格表述
- 更倾向给出保守、低调的评价

- ✅ 评语更严厉、更挑剔
- ✅ 给高分（90+）的概率大幅降低

## 📈 严格度调节参考

如果你觉得 `top-p: 0.3` 还不够严格或太严格，可以调整：

### 更严格（慎用）
```yaml
top-p: 0.1  # 极致严格，可能过于苛刻
```

### 略微放松
```yaml
top-p: 0.5  # 非常严格，但不会过分
```

### 平衡模式
```yaml
top-p: 0.7  # 适度严格
```

## 💡 使用建议

### 何时使用 top-p: 0.3（极度严格）

✅ **推荐场景：**
- 黑客松决赛阶段，需要严格筛选
- 对项目质量要求极高
- 需要区分出真正优秀的项目
- 避免分数膨胀

❌ **不推荐场景：**
- 初赛阶段（会打击参赛者积极性）
- 教学性质的比赛
- 需要鼓励创新和尝试的场景
